use cases

A number of use cases can make building an EPM prototype practical. Prototypes with limited functionality [a] give Users the ability to “touch” the app to be built, as well as prompt important additions to the requirements list, [b] give Project stakeholders an opportunity to more clearly think through the project scope as well as the reasonableness of transformation goals, and [c] give IT team members insight into the database and integration requirements, and provide a head-start on IT resource planning both for the project and post-go-live recurring maintenance

Digital Finance has many definitions. Here are some characteristics of the future finance department:- [a] The use of paper is minimal, [b] Employees use cloud-based apps on mobile devices to execute their tasks, [c] Supported by highly standardized, workflow-enabled business processes, and [d] Finance managers receive event-driven, real-time updates. The Cloud

The benefits of transitioning to a cloud-based, microservices architecture

With micro-services, developers and administrators [a] can Monitor and replace a non-performing software service without impacting any other applications since each micro-service is autonomous and independent. [b] Companies can selectively invest in reusable building code blocks - libraries, modules, and special-purpose applications to compose new and distinct services for a variety of use cases. [c] Unlike monolithic applications, micro-service applications can be selectively scaled out, when the loads shift. This delivers better value from the underlying infrastructure as the need to provision new virtual machines shifts to provisioning new micro-service instances on existing virtual machines. [d] Developers and administrators will be able to opt for best-of-breed technologies that work best with a specific micro-service. They will be able to mix and match a variety of operating systems, languages, frameworks, runtimes, databases and monitoring tools.

  • Category: Cloud computing, Microservices
